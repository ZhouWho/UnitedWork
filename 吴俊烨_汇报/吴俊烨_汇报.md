### 2026.4.14

#### 一、往返播放机制实现
- 增加了 `loop_playback` 开关，支持在指定帧范围内往返播放
- 增加 `frame_start` 和 `frame_end` 参数，可自由选择需要的训练片段，为避免首尾帧不一致导致突变，改为往返播放

#### 二、奖励函数重构与权重调优
关节位置奖励改为指数形式：`Mathf.Exp(-5f * jointError²)`，使奖励更平滑。新增关节速度奖励、动作平滑奖励，减少关节抖动。
| 权重项 | 数值 | 作用 |
|--------|------|------|
| w_pose | 0.45 | 关节位置精度 |
| w_vel | 0.15 | 关节速度匹配 |
| w_root_rot | 0.12 | 身体朝向 |
| w_root_pos | 0.08 | 身体位置 |
| w_upright | 0.10 | 保持直立 |
| w_action_smooth | 0.05 | 动作平滑 |
| w_alive | 0.05 | 存活鼓励 |

#### 三、随机扰动系统实现
- 添加 `add_disturbance` 开关，训练时随机施加外力，增加抗干扰能力
- 扰动参数：力 5-20N，间隔 2-5 秒随机
- if (add_disturbance)//if (train && add_disturbance && !replay)

#### 四、终止条件优化与观测空间完善
- 终止条件放宽：高度 < 0.45m、倾斜 > 80°、位置误差 > 1.5m

### 2026.4.15

#### 键盘控制与动作切换系统实现

**一、功能概述**
- 初步实现了通过键盘按键（W 键）控制机器人切换动作模式
- 目前仅支持站立模式与行走模式的实时切换
- 实现了模型文件与 CSV 数据的联动切换

**二、核心功能实现**

| 功能模块 | 实现方式 | 作用 |
|----------|----------|------|
| 按键检测 | Update() 中检测 W 键输入 | 实时响应玩家控制 |
| 模式切换 | SetIdleMode() / SetWalkMode() | 切换站立/行走状态 |
| 模型切换 | SwitchModel() | 加载对应的神经网络模型 |
| 数据切换 | LoadCSVData() | 加载对应的参考动作数据 |
| 帧范围设置 | 动态修改 frame_start/frame_end | 指定动作播放区间 |
| 姿态对齐 | TeleportToTargetPose() | 切换时保持姿态连续 |

**三、测试结果**
[训练结果视频](./videos/切换结果.mp4)
- W 键按下时：机器人切换为行走模型，播放行走动作（视频中因为还未训练前进所以会倒下）
- W 键松开时：机器人切换回站立模型，保持静止
