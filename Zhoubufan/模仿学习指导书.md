# 1.环境搭建和使用

## 1.1 unity安装与使用

请参考链接 [格物平台readme]([Unity-RL-Playground/README1.md at main · loongOpen/Unity-RL-Playground · GitHub](https://github.com/loongOpen/Unity-RL-Playground/blob/main/README1.md)) 中的入门指南：仿真环境设置

1. 安装Unity Hub并登录。在“安装Unity编辑器”弹窗中，点击跳过，然后选择同意，即可获得个人版许可证，免费激活。
2. 在 Unity Hub 中，打开安装标签→安装编辑器→选择 Unity Editor 2022 （2022.3）（~7 GB，需要耐心等待）。
3. 下载Unity RL Playground：https://github.com/loongOpen/Unity-RL-Playground 并在本地解压。
4. 在Unity Hub中，进入“项目→打开”，然后选择Unity-RL-Playground/gewu文件夹。第一次装载可能需要一段时间;如果被提示，选择忽略或继续。

每个演示场景也可以直接打开，无需使用主菜单。

## 1.2 anaconda安装与简单使用

请参考链接 [格物平台readme]([Unity-RL-Playground/README1.md at main · loongOpen/Unity-RL-Playground · GitHub](https://github.com/loongOpen/Unity-RL-Playground/blob/main/README1.md))：培训环境设置

1. 安装Anaconda：https://www.anaconda.com/download

2. 从开始菜单打开Anaconda提示词。

3. 创造一个新环境：

   `conda create -n gewu python=3.10.12 -y`

4. 激活环境：

   `conda activate gewu`

5. 安装PyTorch（支持CUDA 12.1）：

   `pip3 install torch~=2.2.1 --index-url https://download.pytorch.org/whl/cu121`

（确保网络畅通无阻。这会花很长时间，请耐心等待。如果安装失败，试试其他网络。）

1. 安装机器学习代理：

   `python -m pip install mlagents==1.1.0`

2. 确认安装：`mlagents-learn --help`

注意事项：**确保每次训练前在训练的文件目录打开并激活gewu环境**



# 2 训练方法和步骤

1.打开格物平台后，打开Imitation文件夹便到了我们的模拟学习环境中。双击下方文件中的**G1**的unity环境，打开G1 unity环境。

2.在右侧的inspector，先检查脚本顺序：从上到下分别是：



**G1mimic 1 Agent训练脚本**：前四个分别是训练模式，播放模式，学习对象的id，学习对象的名字，我们主要修改前三个：

​	**Train模式**：在使用anaconda进行训练时点击勾选，**确保取消Replay模式**。

​	**Replay模式**：可强制提供csv脚本的关节角度，使机器人忽视物理环境进行播放。

​	**Motion_id**：填写要参考的文件id，在\Assets\StreamingAssets\g1_dataset中，第一个文件是id0，以此类推到要参考训练的文件id

​	**当然，脚本内部的奖励函数等需要根据具体的任务进行修改！**



**Behavior Parameters 强化学习脚本**：我们主要修改Space Size，Stacked Vectors和Model：

​	**Space Size**：默认为68个。

​	**Stacked Vectors**：改为1。

​	**Model**：拖拽你训练好的模型（在results文件中）并取消勾选agent脚本的train和replay查看训练效果

**Decision Requestor脚本**：不作赘述。

![inspector](.\image\inspector.png)

3.检查完所有设置好的文件参数后，windows系统打开anaconda后，输入`cd \当前文件夹路径`

进入当前文件夹，另外`cd ..`是返回上一级文件层。进入文件夹后使用`conda activate gewu`激活gewu的训练环境，然后输入  

` mlagents-learn config.yaml --run-id=go2trot --force`

run-id=后面的字符是你要训练的模型名字，--force是从头开始训练，如果中途要继续训练则用

` mlagents-learn config.yaml --run-id=go2trot --resume`

输入后等命令行出现让你play的字样时，在unity中按开始键进行训练。

# 3 任务分工---第五周至第六周任务

​	我们的任务目标是完成 **通过模拟学习实现一个宇树G1人形机器人的拳击擂台赛**

所以我们需要完成的大概**前期**步骤是：



## 3.0 进行csv的数据划分（1到2人）：

对存放在\Assets\StreamingAssets\g1_dataset的fight拳击motion_id数据csv进行数据分割，我们需要分割出两类数据：

1.用于训练模型的完全覆盖动作数据：比如说左勾拳在csv文件的2000帧-2200帧完成，我们就需要分割出1700-2500帧来进行训练，确保有前后动作的过渡，以确保模型最终训练有自然的动作过渡功能。

2.真实数据的分割：这一部分的数据是用于后续训练完模型需要基于左勾拳的动作调用该动作数据的时候所用。



简单来说：我们的机器人依照大范围数据进行学习生成模型，随后模型再根据小范围精确动作数据输出我们想要的动作，所以我们需要两类数据的分割。

**目前我们需要的数据有：左勾拳，右勾拳，左踢腿，右踢腿，后续可能会有起身等动作的分割**。



**liuxu于4.1号已经提交**

   的报错，需要在G1mimic1Agent中做如下修改
   使用 取模运算 (%)。这样当 currentFrame 超过最大索引时，会自动回到 0。

   修改 OnActionReceived 里的代码：
   ```bash
   C#
   if (itpData.Count > 0)
   {
      // 使用 % 确保索引永远在 0 到 Count-1 之间
      currentData = itpData[currentFrame % itpData.Count]; 
      
      Array.Copy(currentData, 0, currentPos, 0, 3);
      // ... 后续代码不变
   }




   OnEpisodeBegin 里的这一行：

   C#
   if(rand) frame0 = 1000 + Random.Range(0,5)*100;
   currentFrame = frame0;
   如果CSV 数据文件总长度小于 1500 帧（1000 + 500），这段代码在启动的一瞬间就会崩溃。

   建议修复：
   确保初始帧也在数据范围内。

   C#
   currentFrame = frame0 % itpData.Count;
   ```
